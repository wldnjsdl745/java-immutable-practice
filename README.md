# java-immutable-practice
Java에서 **가변 객체와 불변 객체**를 비교하며 **Immutable Class**를 연습한 프로젝트

## 학습 내용
- 가변 객체와 불변 객체의 차이
- 공유 참조로 인한 사이드 이펙트
- 불변 클래스 설계
- 객체 상태 변경 대신 새로운 객체 생성

## 기술 스택
- Java

## 핵심 개념
- 가변 객체는 객체 생성 이후 내부 상태를 변경할 수 있는 객체입니다.
- 가변 객체를 공유 참조하면 한 곳의 변경이 다른 참조에도 영향을 줄 수 있습니다.
- 불변 객체는 객체 생성 이후 내부 상태를 변경할 수 없도록 설계한 객체입니다.
- 상태를 변경해야 할 경우 기존 객체를 수정하지 않고 새로운 객체를 생성해야 합니다.
